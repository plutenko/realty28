# Система автоматического распределения лидов

## Модуль для платформы «СОБР» — отдел продаж новостроек

---

## Проблема

Сейчас заявки от потенциальных покупателей поступают через различные источники (квизы, лендинги, звонки), и руководитель продаж вручную распределяет их между риелторами. Это создаёт задержки: пока руководитель увидит заявку, пока назначит — клиент остывает. Исследования показывают, что вероятность успешного контакта падает в 10 раз, если перезвонить не за 5 минут, а за 30.

---

## Решение

Автоматическая система распределения лидов, интегрированная в существующую платформу «СОБР». Заявки из любых источников мгновенно поступают назначенным риелторам через Telegram-бот. Первый, кто нажимает «Беру в работу», получает контакт клиента и начинает работу — без участия руководителя.

---

## Как это работает

### Шаг 1. Поступление заявки

Клиент оставляет заявку на одном из подключённых источников:

| Источник | Как подключается |
|---|---|
| Марквиз (квизы) | Webhook — URL из системы вставляется в настройки Марквиза |
| Тильда (лендинги) | Webhook — URL вставляется в настройки формы Тильды |
| Ручной ввод | Руководитель добавляет лид вручную через личный кабинет |
| Другие CRM/сервисы | Подключаются по аналогии — один новый адаптер на источник |

### Шаг 2. Мгновенное уведомление

Все назначенные риелторы (например, 7 из 20) одновременно получают уведомление в Telegram:

```
📥 Новая заявка

Источник: Марквиз — квиз подбор квартиры
Имя: Алексей
Бюджет: 3–5 млн
Комнат: 2
Время: 15:32

[🔥 Беру в работу]   [⏭ Пропускаю]
```

Важно: телефон и email клиента НЕ показываются до нажатия кнопки — это исключает работу «мимо системы».

### Шаг 3. Первый забирает

Риелтор нажимает «🔥 Беру в работу» прямо в Telegram. Происходит:

**Ему** — приходит сообщение с контактами:
```
✅ Вы взяли заявку

📞 +7 900 123-45-67
📧 alex@mail.ru

Позвоните клиенту в ближайшие 5 минут.
```

**Остальным 6 риелторам** — сообщение обновляется:
```
🔒 Заявку взял Иван (15:33, через 47 сек)
```
Кнопки исчезают.

**Руководителю** — приходит уведомление в Telegram:
```
✅ Заявку взял Иван

Источник: Марквиз — квиз подбор квартиры
Клиент: Алексей
Время реакции: 47 сек
```
Также запись появляется в дашборде: кто взял, когда, время реакции.

### Шаг 4. Работа с лидом

Риелтор ведёт клиента по воронке, обновляя статус:

```
Новый → В работе → Позвонил → Встреча → Показ → Бронь → Сделка
                                                      → Отказ (причина)
```

Статус можно менять через Telegram-бот или в личном кабинете на сайте.

### Шаг 5. Контроль руководителя

Руководитель видит полную картину в дашборде без необходимости участвовать в рутине:

- Все лиды из всех источников в одной таблице
- Кто из риелторов сколько взял
- Среднее время реакции по каждому
- Конверсия: заявки → сделки
- Незабранные и просроченные заявки

## Защита от потери заявок

| Ситуация | Что происходит |
|---|---|
| Никто не нажал «Беру» за 5 минут | Руководителю приходит уведомление: «⚠ Заявку никто не взял!» с кнопкой ручного назначения |
| Риелтор взял, но не позвонил за 10 минут | Напоминание риелтору + уведомление руководителю |
| Повторная заявка с того же номера | Система распознаёт дубль: «Повторная заявка от Алексея, в работе у Ивана» — не рассылает заново |
| У риелтора уже 10 активных лидов | Ему кнопка «Беру» недоступна, пока не закроет текущие |

---

## Настройка источников

Руководитель в личном кабинете создаёт источники лидов:

1. Нажимает «+ Добавить источник»
2. Указывает название: «Марквиз — квиз подбор квартиры»
3. Выбирает тип: Марквиз / Тильда / Ручной ввод
4. Назначает риелторов-получателей (чекбоксы)
5. Получает уникальный URL вебхука
6. Вставляет этот URL в настройки Марквиза или Тильды

Источников может быть сколько угодно. У каждого свой список риелторов. Например:
- «Квиз — подбор квартиры» → 7 риелторов отдела новостроек
- «Лендинг — ЖК Сити» → 3 риелтора, специализирующихся на этом ЖК
- «Звонки в офис» → все 20 риелторов

## Технические детали

### Что используется

| Компонент | Технология | Стоимость |
|---|---|---|
| Бэкенд + API | Vercel (уже используется) | Бесплатно |
| База данных | Supabase (уже используется) | Бесплатно |
| Telegram-бот | Telegram Bot API (бот уже создан) | Бесплатно |
| Фронтенд | Next.js (уже используется) | Бесплатно |

Дополнительных платных сервисов не требуется. Модуль встраивается в существующую платформу «СОБР».

### Ёмкость

- База данных Supabase (500 MB) вмещает 250 000+ лидов
- При 50 лидах в день — запас на 10+ лет
- Telegram Bot API — без лимитов на количество сообщений
- Vercel — 100 000 вызовов/месяц, расход ~15 000 при 50 лидах/день

### Безопасность

- Телефон клиента видит только тот риелтор, который взял заявку
- Вебхуки защищены секретным ключом — посторонний не сможет отправить фейковую заявку
- Все действия логируются: кто взял, когда, что делал
- Данные хранятся в Supabase с шифрованием и row-level security

## Этапы внедрения

### Этап 1. Базовый модуль
- Таблицы в базе данных
- Приём заявок через вебхуки (Марквиз + Тильда)
- Рассылка в Telegram с кнопками
- Обработка «Беру в работу»
- Таблица лидов в дашборде руководителя

### Этап 2. Расширение
- Ручной ввод лидов
- Настройка источников через интерфейс
- Назначение риелторов по источникам
- Автоэскалация (уведомление при отсутствии реакции)

### Этап 3. Аналитика
- Воронка продаж по статусам
- Метрики по риелторам (время реакции, конверсия)
- Сводные отчёты за период
- Распознавание дублей по номеру телефона

## Преимущества

**Для руководителя:**
- Заявки распределяются автоматически — не нужно сидеть и назначать вручную
- Полный контроль: видно кто взял, как быстро, какой результат
- Ни одна заявка не потеряется — система следит за таймаутами
- Объективные метрики для оценки эффективности каждого риелтора

**Для риелторов:**
- Моментальное получение заявок в Telegram — не нужно проверять почту или CRM
- Справедливое распределение — кто быстрее реагирует, тот больше зарабатывает
- Минимум действий: одна кнопка — и контакт клиента у тебя

**Для бизнеса:**
- Скорость реакции: от заявки до звонка — меньше минуты вместо часов
- Прозрачность: каждый лид отслеживается от заявки до сделки
- Масштабируемость: подключение нового источника — 5 минут
- Нулевая стоимость инфраструктуры — всё на бесплатных планах
